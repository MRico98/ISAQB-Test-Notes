# Requriiments and best prectices for designinig good interfaces

## Requiriments of a Good interface
* Fullfill user reqeuiriments
    * Suitable functionality

* Easy to learn and to use
    * Provide IDL
    * Use unit test as documentation

* Easly to extend
    * Consider API Version

* Hard to misuse
* Consistenti to ther interfaces in the system

## Best practices
* Use it yourself, implemnt tsts, and use the test as a reference example
* Clarify requiriments with the consumer and provider
* Hide implementation details
* Use meaningul, self descriptive names
* Service oriented interface operation
* Minize surpises and side effects
* Provide atomic inforamtion units
* Documentation
